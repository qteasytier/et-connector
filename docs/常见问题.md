# 常见问题

## 1. 服务启动失败

### 问题现象
- 点击"启动连接"后显示"服务启动失败"
- 系统托盘图标显示红色状态

### 可能原因和解决方法

| 原因 | 解决方法 |
|------|----------|
| `easytier-cli.exe` 不存在 | 确认 `etcore/easytier-cli.exe` 文件存在且可执行 |
| 连接密钥无效 | 检查密钥是否正确，重新输入 |
| UAC 授权被取消 | 重新启动程序，再次尝试启动 |
| 依赖文件缺失 | 确保 `etcore/` 目录下的所有文件完整 |
| 端口被占用 | 检查 15888 端口是否被其他程序占用 |

### 排查步骤
1. 检查 `etcore/easytier-cli.exe` 是否存在
2. 确认连接密钥正确
3. 查看事件查看器中的错误信息
4. 尝试手动运行 `easytier-cli.exe service install`

## 2. 图标不显示

### 问题现象
- 系统托盘没有图标显示
- 程序在后台运行但无法看到

### 可能原因和解决方法

| 原因 | 解决方法 |
|------|----------|
| 图标文件缺失 | 确保 `assets/favicon.ico` 存在 |
| 系统托盘被禁用 | 检查 Windows 系统托盘设置 |
| 程序崩溃 | 查看事件查看器中的错误日志 |

### 排查步骤
1. 检查 `assets/favicon.ico` 文件是否存在
2. 重启程序
3. 检查 Windows 系统托盘设置

## 3. 程序提示"已在运行"

### 问题现象
- 启动程序时显示"EasyTier 控制台连接器已在运行"
- 无法启动第二个实例

### 原因
- 基于单实例检测机制，防止多个实例同时运行

### 解决方法
1. 关闭已运行的实例
2. 或使用命令行参数 `--force-start` 强制启动

## 4. 修改密钥后无法连接

### 问题现象
- 修改密钥后，服务无法启动
- 显示"服务启动失败"

### 原因
- 旧服务仍在运行，新密钥未生效

### 解决方法
1. 停止当前服务（右键菜单 → 停止连接）
2. 等待 500ms
3. 使用新密钥重新启动

## 5. 开机自启失效

### 问题现象
- 勾选了"开机启动托盘"但程序未自动启动

### 可能原因和解决方法

| 原因 | 解决方法 |
|------|----------|
| 注册表项未正确写入 | 手动检查 `HKCU\Software\Microsoft\Windows\CurrentVersion\Run` |
| 程序安装路径包含空格 | 确保程序路径正确格式化 |
| 系统策略限制 | 检查组策略中的自启限制 |

### 排查步骤
1. 检查注册表项：`HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run\EasyTierConnector`
2. 确认值数据格式正确：`"C:\Program Files\EasyTier\Connector\EasyTierConnector.exe" --auto-start`
3. 检查 Windows 启动文件夹

## 6. 连接状态异常

### 问题现象
- 状态显示为"已连接"但无法访问 Web 控制台
- 状态显示为"连接中"但长时间无变化

### 可能原因和解决方法

| 原因 | 解决方法 |
|------|----------|
| 网络连接问题 | 检查网络连接 |
| 防火墙阻止 | 检查防火墙设置 |
| EasyTier Core 服务异常 | 重启服务 |
| 端口被占用 | 检查 15888 端口 |

### 排查步骤
1. 检查 `easytier-core.exe` 进程是否运行
2. 查看 `etcore/easytier.log` 日志文件
3. 检查端口占用情况：`netstat -ano | findstr 15888`
4. 重启 EasyTier Core 服务

## 7. 程序崩溃

### 问题现象
- 程序突然关闭
- 系统托盘图标消失

### 可能原因和解决方法

| 原因 | 解决方法 |
|------|----------|
| 内存不足 | 关闭其他程序释放内存 |
| 依赖文件损坏 | 重新安装程序 |
| 系统兼容性问题 | 以兼容模式运行 |

### 排查步骤
1. 查看 Windows 事件查看器中的错误日志
2. 检查 `etcore/easytier.log` 日志文件
3. 重新安装程序

## 8. 卸载问题

### 问题现象
- 卸载程序后仍有残留
- 注册表项未完全清除

### 解决方法
1. 手动删除注册表项：`HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run\EasyTierConnector`
2. 删除程序安装目录
3. 重启计算机

## 9. 性能问题

### 问题现象
- 程序占用过高 CPU 或内存
- 系统响应变慢

### 可能原因和解决方法

| 原因 | 解决方法 |
|------|----------|
| 心跳检测频率过高 | 调整 `heartbeatInterval` 配置 |
| 服务启动超时 | 调整 `serviceTimeout` 配置 |
| 资源竞争 | 关闭其他占用资源的程序 |

### 优化建议
1. 增加 `heartbeatInterval` 到 5000ms
2. 减少 `maxReconnectAttempts` 到 3
3. 监控系统资源使用情况

## 10. 跨版本升级

### 升级步骤
1. 备份当前配置文件：`%LOCALAPPDATA%/EasyTier/config.json`
2. 卸载旧版本
3. 安装新版本
4. 恢复配置文件
5. 启动程序

### 注意事项
- 升级前备份重要数据
- 检查新版本的系统要求
- 测试功能是否正常
