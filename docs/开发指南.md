# 开发指南

## 1. 项目结构

```
et-web-connector/
├── src/                      # 源代码
│   ├── main.cpp              # 程序入口，命令行解析，单实例检测
│   ├── SystemTray.h/cpp      # 系统托盘主类
│   ├── ETRunService.h/cpp    # EasyTier 系统服务管理器（静态类）
│   ├── ConfigManager.h/cpp   # JSON 配置文件管理
│   ├── SettingsDialog.h/cpp  # 连接密钥设置对话框
│   └── AboutDialog.h/cpp     # 关于软件对话框
├── assets/                   # 图标资源 (SVG/ICO)
├── etcore/                   # EasyTier Core 依赖
│   ├── easytier-core.exe     # 核心程序
│   ├── easytier-cli.exe      # 命令行管理工具
│   ├── wintun.dll            # Wintun 驱动
│   ├── Packet.dll            # 网络包处理
│   └── WinDivert64.sys       # WinDivert 驱动
├── resources.qrc             # Qt 资源文件定义
├── app.manifest              # Windows 应用清单 (UAC/DPI/兼容性)
└── CMakeLists.txt            # CMake 构建配置
```

## 2. 开发环境设置

### 2.1 环境要求
- Qt 6.11 或更高版本
- CMake 3.16+
- Visual Studio 2019/2022 或 Qt Creator
- llvm-mingw 或 MSVC 编译器

### 2.2 环境配置

#### Qt Creator
1. 打开 Qt Creator
2. 选择"文件" → "打开文件或项目"
3. 选择 `CMakeLists.txt`
4. 配置 Kit：选择 Qt 6.11 和编译器
5. 构建项目

#### Visual Studio
1. 打开 Developer Command Prompt
2. 运行 `cmake -S . -B build -G "Visual Studio 17 2022" -DCMAKE_PREFIX_PATH=C:/Qt/6.11.1/msvc2019_64`
3. 打开 `build/EasyTierConnector.sln`
4. 构建解决方案

## 3. 代码规范

### 3.1 命名规范
- 类名：PascalCase（如 `SystemTray`）
- 变量名：camelCase（如 `connectionKey`）
- 常量：UPPER_SNAKE_CASE（如 `SERVICE_NAME`）
- 成员变量：`m_` 前缀（如 `m_configManager`）

### 3.2 注释规范
- 使用 Doxygen 风格注释
- 公共接口必须有详细注释
- 复杂逻辑需要注释说明

### 3.3 代码风格
- 使用 Qt 信号槽新语法
- 避免使用 `using namespace std;`
- 使用 RAII 管理资源
- 异常处理使用 try-catch

## 4. 核心模块开发

### 4.1 SystemTray 类

#### 职责
- 管理系统托盘图标和菜单
- 协调各模块间的信号槽
- 处理用户交互

#### 主要方法
```cpp
// 初始化系统托盘
void SystemTray::setupMenu();

// 更新连接状态
void SystemTray::updateStatus(ConnectionState state);

// 处理托盘激活事件
void SystemTray::onTrayActivated(QSystemTrayIcon::ActivationReason reason);

// 处理连接密钥变更
void SystemTray::onConnectionKeyChanged();
```

### 4.2 ETRunService 类

#### 职责
- 管理 EasyTier Core 系统服务
- 提供静态方法执行命令
- 处理 UAC 提权

#### 主要方法
```cpp
// 启动服务
static bool start(const QString &connectionKey);

// 停止并卸载服务
static bool stop();

// 检查服务是否运行
static bool isRunning();

// 执行命令并等待完成
static CommandResult executeCommand(const QString &command, 
                                  const QStringList &args,
                                  int timeoutMs = 30000);
```

### 4.3 ConfigManager 类

#### 职责
- 管理 JSON 配置文件
- 提供配置读写接口
- 处理配置迁移

#### 主要方法
```cpp
// 加载配置
bool loadConfig();

// 保存配置
void saveConfig();

// 获取连接密钥
QString getConnectionKey();

// 设置连接密钥
void setConnectionKey(const QString &key);
```

## 5. 构建和部署

### 5.1 构建命令
```bash
# 创建构建目录
mkdir build && cd build

# 配置 CMake
cmake .. -DCMAKE_PREFIX_PATH=/path/to/Qt6.11

# 编译
cmake --build . --config Release

# 安装
cmake --install .
```

### 5.2 部署
- 使用 `windeployqt` 部署 Qt 运行时
- 确保 `etcore/` 目录完整
- 包含所有依赖文件
